## Overview

The `Input` component is a customizable text input field used to capture user data such as text, numbers, or passwords. It supports various input types and states like disabled or read-only.

### Features:
- Supports multiple input types (e.g., text, password, email).
- Can be styled for different states (e.g., disabled, focused).
- Customizable placeholder text and validation.

## Props

| Prop        | Type       | Description                                                         |
| ----------- | ---------- | ------------------------------------------------------------------- |
| `type`      | `string`   | Specifies the input type (e.g., "text", "password", "email").        |
| `placeholder`| `string`  | Placeholder text displayed inside the input field.                   |
| `value`     | `string`   | The current value of the input.                                      |
| `onChange`  | `function` | Callback function triggered when the input value changes.            |
| `disabled`  | `boolean`  | Disables the input field if true.                                    |
| `className` | `string`   | Optional additional classes for styling the input field.             |
